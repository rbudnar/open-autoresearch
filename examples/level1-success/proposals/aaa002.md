# Experiment Proposal: Ordinal CE hybrid loss

## Hypothesis

Because the baseline's NLL on tail-ordered classes is high while accuracy on the modal class is fine, switching to an ordinal-aware hybrid loss (`ordinal_ce_hybrid`) should reduce NLL by penalizing far-off mispredictions more strongly, without hurting accuracy or latency.

## Literature basis

Offline mode (`mode: offline`, `web_search_used: false`). Reference from `canon.bib`: Karpathy autoresearch baseline notes that ordinal loss variants are an obvious early branch for classification with structured target space. No novel-architecture claims (offline mode blocks those per §9.0).

## Proposed change

Swap the cross-entropy loss for an ordinal-aware variant in `losses/ordinal_ce_hybrid.py`. Wire `loss_type: ordinal_ce_hybrid` into the training config. No other config switches change.

## Single non-baseline switch

```yaml
config:
  loss_type: ordinal_ce_hybrid
# all other switches pinned to baseline
```

## Evaluation plan

- **Smoke (Stage A):** 100 steps on 64 examples. Verify loss finite, output shape correct.
- **Proxy (Stage B):** 5% train split, separate from val. ~10 min.
- **Full validation (Stage C):** 3 seeds (small tier).
- **Ablation type:** Not at Level 1.

## Early stopping rule

Stop if proxy NLL after 1 epoch is > baseline proxy NLL + 0.05 (clearly dominated per §12.2.1).

## Risks

- **Leakage:** Loss touches only training-time logic; no test/val data touched.
- **Compute:** Within max_gpu_hours_per_candidate = 3.
- **Metric gaming:** Ordinal weighting can shift the model toward modal predictions; check per-class behavior in the report.
- **Implementation:** Hybrid coefficient hardcoded to 0.5; sensitivity sweep deferred to Level 3.
